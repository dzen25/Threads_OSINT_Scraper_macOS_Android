# 🧵 Threads XML Scraper (ADB Edition)

**Threads XML Scraper** — это легковесный и высокоточный инструмент для OSINT-исследований и парсинга ленты Threads без использования OCR (распознавания текста) или официальных API. 

Инструмент извлекает данные напрямую из системной иерархии Android (UI XML), что гарантирует 100% точность текста и нулевую нагрузку на процессор вашего Mac/PC.

---

## 🚀 Основные возможности
* **Direct Parsing:** Никакого размытого текста — данные берутся из цифрового дампа экрана.
* **No-AI Speed:** Работает мгновенно, не требуя сторонних нейросетей для извлечения текста.
* **Wireless:** Полная поддержка отладки по Wi-Fi (свобода от кабелей).
* **Auto-Scroll:** Автоматический бесконечный свайп с контролем дублей.
* **Analytics Ready:** Сохраняет данные в чистый JSON для последующего анализа.

## ⚙️ Настройка Android
Для работы скрипта ваш смартфон должен быть доступен через ADB.

1.  **Режим разработчика:** Перейдите в *Настройки -> О телефоне* и нажмите на *Номер сборки* 7 раз.
2.  **USB Отладка:** Включите «Отладка по USB» в меню для разработчиков.
3.  **Беспроводная отладка (рекомендуется):**
    * Подключите телефон кабелем к Mac/PC.
    * В терминале: `adb tcpip 5555`.
    * Узнайте IP телефона (Настройки Wi-Fi) и введите: `adb connect <IP_АДРЕС>:5555`.
    * Теперь кабель можно отключить.
4.  **Stay Awake:** Чтобы экран не тух во время сбора данных:
    ```bash
    adb shell settings put system screen_off_timeout 86400000
    ```
---    
## Для того чтобы скрипт пролистывал ровно один экран, нужно адаптировать координаты свайпа.

#### 5. Узнайте разрешение экрана
Выполните команду:
```bash
adb shell wm size
```
*Пример вывода: `Physical size: 1080x2160`*

#### Рассчитайте координаты
В коде используется функция `subprocess.run(["adb", "shell", "input", "swipe", "X1", "Y1", "X2", "Y2", "Duration"])` - вам нужно найти эту строку в файле `main.py` и изменить значения на свои.

1.  **X1 и X2 (Центр):** Разделите ширину экрана на 2. (Для 1080 это **540**).
2.  **Y1 (Старт - низ):** Возьмите высоту экрана и отнимите ~10-15%. (Для 2160 это примерно **1900**).
3.  **Y2 (Финиш - верх):** Установите значение около **200**, чтобы не задевать шторку уведомлений.
4.  **Duration (Скорость):** Оптимально **500** мс.

**Пример для 1080x2160:**
`subprocess.run(["adb", "shell", "input", "swipe", "540", "1900", "540", "200", "500"])`

## 🐍 Подготовка окружения

### Нужно ли виртуальное окружение?
**Да.** Рекомендуется использовать `venv`, чтобы изолировать зависимости проекта и избежать конфликтов с системными библиотеками Python.

### Установка и запуск
1.  **Создайте окружение:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate  # для macOS/Linux
    # venv\Scripts\activate   # для Windows
    ```
2.  **Зависимости:**
    Скрипт использует стандартные библиотеки Python (`json`, `xml`, `subprocess`, `time`), поэтому дополнительные `pip install` не требуются. Убедитесь, что у вас установлен **ADB** (Android SDK Platform-Tools).

## 🛠 Запуск скрипта

### Основной сборщик (main.py)
Запускает бесконечный цикл: Дамп экрана -> Парсинг -> Свайп.
```bash
python3 main.py
```
* Данные сохраняются в файл `threads_base.json`.
* Скрипт автоматически проверяет дубликаты по тексту поста, чтобы не записывать одно и то же дважды.

### Поиск повторов (analyze_users.py)
Инструмент для анализа активности аудитории в собранной базе.
```bash
python3 find_replay_user.py
```
**Инструкция:**
1. Запустите скрипт после того, как соберете хотя бы 100-200 записей.
2. Он выведет таблицу пользователей, которые чаще всего встречаются в ленте.
3. Это поможет выявить "лидеров мнений" или наиболее активных ботов в вашей выборке.

### 💡 Совет по эксплуатации
Если вы заметили, что посты начинают "пролетать" мимо парсера, увеличьте `time.sleep()` в `main.py` до 2-3 секунд. Это даст вашему смартфону больше времени на прогрузку тяжелого интерфейса Threads.
