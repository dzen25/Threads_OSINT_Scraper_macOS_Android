# 🕵️‍♂️ Threads OSINT Scraper (macOS + Android)

Автоматизированный скрипт для сбора данных из социальной сети **Threads** (и не только - достаточно изменить промт внутри кода на свой). Использует мощь нативного **Apple Vision Framework** для мгновенного OCR и локальные LLM (через LM Studio) для интеллектуального анализа профилей.

## ✨ Особенности
- **Молниеносный OCR:** Работает на базе Apple Silicon (M1, M2, M3) через системные API macOS.
- **Умный анализ:** Не просто копирует текст, а вычленяет факты (хобби, локации, питомцы) и определяет "вайб" поста.
- **Автоматизация:** Сам листает ленту и аккумулирует данные в один структурированный JSON.
- **Zero-Footprint:** Минимум зависимостей, никакой тяжелой нейронки EasyOCR или PyTorch на диске.

---

## 🛠 Требования
1. **Железо:** Mac с процессором **Apple Silicon (M-серия)**.
2. **ОС:** macOS (для работы Vision Framework).
3. **Android:** Телефон с включенной "Отладкой по USB".
4. **LM Studio:** С загруженной моделью (рекомендуется `Qwen 2.5 7B Instruct`).

---

## 🚀 Быстрый старт

### 1. Подготовка окружения
Откройте терминал в папке с проектом:


#### Создаем виртуальное окружение
```bash
python3 -m venv venv
```
#### Активируем его
```bash
source venv/bin/activate
```
#### Устанавливаем необходимые пакеты
```bash
pip install pure-python-adb Pillow requests pyobjc-framework-Vision pyobjc-framework-Quartz
```
---

### 2. Настройка LM Studio
1. Запустите **LM Studio**.
2. Загрузите и выберите модель (например, `qwen2.5-7b-instruct`).
3. Перейдите во вкладку **Local Server** и нажмите **Start Server**.
4. Убедитесь, что сервер доступен по адресу `http://localhost:1234`.
- Если ваши данные инные то вам необходимо изменить значение переменных в файле `main.py`
  
`URL = "http://127.0.0.1:1234/v1/chat/completions"`

`MODEL_NAME = "qwen2.5-7b-instruct"`

---

### 3. Подключение телефона
1. Подготовка Android (Без Root)
2. Включите Настройки разработчика (7 кликов по "Версии сборки").
3. Включите Отладку по USB.

- Для Xiaomi/Poco/Redmi (Важно): Включите пункт "Отладка по USB (Настройки безопасности)". Без этого свайпы через код работать не будут.
  
4. Подключите Android-смартфон к Mac кабелем.
5. В терминале выполните: `adb devices`. Должен появиться идентификатор вашего устройства.

---

### 4. Настройка свайпа под ваш экран

Для того чтобы скрипт пролистывал ровно один экран, нужно адаптировать координаты свайпа.

#### 1: Узнайте разрешение экрана
Выполните команду:
```bash
adb shell wm size
```
*Пример вывода: `Physical size: 1080x2160`*

#### 2: Рассчитайте координаты
В коде используется функция `device.shell("input swipe X1 Y1 X2 Y2 Duration")` - вам нужно найти эту строку в файле `main.py` и изменить значения на свои.

1.  **X1 и X2 (Центр):** Разделите ширину экрана на 2. (Для 1080 это **540**).
2.  **Y1 (Старт - низ):** Возьмите высоту экрана и отнимите ~10-15%. (Для 2160 это примерно **1900**).
3.  **Y2 (Финиш - верх):** Установите значение около **200**, чтобы не задевать шторку уведомлений.
4.  **Duration (Скорость):** Оптимально **500** мс.

**Пример для 1080x2160:**
`device.shell("input swipe 540 1900 540 200 500")`

---
## 5. Запуск скрипта
Подключите телефон и откройте ленту Threads.

Запустите основной файл:
```bash
python main.py
```

## 📂 Структура данных
Результат сохраняется в `settings_test.json`. Скрипт автоматически дополняет существующий файл, не перезаписывая старые данные.

```json
{
    "detected_profiles": [
        {
            "user": "username",
            "post_content": "Текст поста...",
            "personal_intelligence": ["факт 1", "факт 2"],
            "vibe": "настроение"
        }
    ]
}
```

## ⚠️ Важно
При первом запуске macOS может запросить разрешение на доступ к "Записи экрана" или "Автоматизации" для Терминала. **Необходимо подтвердить доступ**, чтобы Apple Vision мог анализировать скриншоты.

## 🛑 Остановка
Чтобы остановить сбор данных, нажмите `Ctrl + C` в окне терминала. Все собранные на этот момент данные будут сохранены в JSON.
